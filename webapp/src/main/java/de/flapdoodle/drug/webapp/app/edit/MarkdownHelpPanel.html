Es wird die Markdown-Syntax mit ein paar Erweiterungen benutzt.
Folgende Erweiterungen können verwendet werden:
  • [[Begriff]] - Erzeugt einen Link auf eine Begriffsdefinition für Begriff
  • [!s3:WortMitBeugung->Wortursprung]
    • Subjekt(s), Prädikat(p) und Objekt(o) - Erzeugt einen Link auf eine Relation die aus Subjekt(s,optional), Prädikat(p) und Objekt(o) bestehen kann.
      Dabei wird versucht, die Zuordnung so gut es get automatisch zu erkennen. Es gibt verschiedene Geltungsbereiche wie z.B. Absätze und Aufzählungen innerhalb deren Grenzen eine Zuordnung versucht wird. Sollte man mehr als einmal ein Subjekt, Prädikat oder Objekt innerhalb eines Geltungsbereichs benutzt haben, ist die Zuordnung nicht möglich (sieht man bei der Darstellung genau).
    • Index(0..9) - Kollisionen können vermieden werden, indem ein zusätzlicher Index angegeben werden kann. Dabei geht der Wertebereich von 0-9. Der Index mit dem Wert 0 entspricht nicht dem Wert, wenn der Index nicht angegeben wird.
    • Sichtbarkeit - Wenn man ein ! angibt, dann wird der Link nicht dargestellt.
    • Wenn ein Wort in gebeugter Form im Text verwendet wird, sollte man die ungebeugte Form für die Relation verwenden. Dazu kann man optional ->Ungebeugt angeben.
    • Es muss nicht immer Subjekt, Prädikat und Objekt angegeben werden. Lässt man Teile weg, werden alle Treffer aufgelistet, die zu den angegeben Teilen passen.
    • Ein paar Beispiele:
      • [s:Wie man->Mensch] ein [o:Loch] [p:bohrt->bohren]
      • Was kann [!s:Mensch] [p:gebohrt->bohren] werden?
  • [!at3:WortMitBeugung->Wortursprung]
    • Kontextbezug mit "Ortsangabe" ( at, from, to, near) - Erzeugt einen Link auf eine Relation die aus Subjekt, Prädikat und Objekt bestehen kann und ergänzt diese um einen Kontext.
    • Ein paar Beispiele:
      • [s:Wie man->Mensch] ein [o:Loch] [p:bohrt->bohren] in der [at:Werkstatt]
      • Was kann [!s:Mensch] in der [at:Werkstatt] [p:gebohrt->bohren] werden?
